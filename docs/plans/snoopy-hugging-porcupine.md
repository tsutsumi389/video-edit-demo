# 動画編集ソフト — 必要機能の洗い出しと実装ロードマップ

## Context

現在の `video-edit-demo` は Electron + React + TypeScript で構築された動画編集アプリで、以下は既に実装済み:

- マルチトラック・タイムライン (`src/renderer/components/Timeline.tsx`)
- クリップの分割 / 削除 / 移動 (`src/renderer/hooks/useProject.ts`)
- Undo / Redo (50段階スタック)
- 動画ファイルのインポート + MP4 エクスポート (`src/main/ffmpeg-service.ts`)
- EDL フラット化 (`src/renderer/utils/flatten.ts`)
- プレビュー再生 (`src/renderer/components/Preview.tsx`)

一方、一般的な動画編集ソフトとして不可欠な機能 (プロジェクト保存、トリミングUI、音声制御、テキスト、エフェクト等) が未実装である。本ドキュメントは **今後実装すべき機能の全体像と優先順位付きロードマップ** を提示する。

## 機能の全体マップ

### A. 基盤 (Foundations)
プロジェクトが壊れずに使い続けられるための土台。

| # | 機能 | 概要 |
|---|------|------|
| A1 | プロジェクト保存 / 読み込み | `.vedit.json` 等で Project 状態を永続化。File メニューから Save / Save As / Open |
| A2 | 最近使ったファイル | 起動時に最近のプロジェクトを提示 |
| A3 | 自動保存 / クラッシュ復旧 | N 分ごとにテンポラリへ書き出し |
| A4 | エラートースト UI | ffmpeg 失敗など現状サイレントな例外を可視化 |

### B. タイムライン / 編集基本操作
編集作業の生産性に直結。

| # | 機能 | 概要 |
|---|------|------|
| B1 | クリップのトリミング UI | クリップ左右端ドラッグで `inPoint` / `outPoint` を変更 (reducer は `TRIM_CLIP` 既存) |
| B2 | タイムラインのズーム | 横方向 px/sec を可変。Ctrl+ホイール |
| B3 | スナップ | クリップ端 / 再生ヘッド / グリッドへの吸着 |
| B4 | リップル削除・挿入 | 削除時に後続クリップを詰める。挿入時にずらす |
| B5 | キーボードショートカット拡充 | Space (再生/停止), J/K/L (逆再生/停止/再生), I/O (In/Out 設定), Home/End |
| B6 | マーカー / チャプター | タイムラインに印を付けて名前をつける |
| B7 | クリップのコピー / ペースト / 複製 | Ctrl+C/V/D |

### C. オーディオ
現在は映像のみを扱っている。

| # | 機能 | 概要 |
|---|------|------|
| C1 | オーディオトラック分離 | `Track` に `kind: 'video' \| 'audio'` を追加 |
| C2 | 波形表示 | ffmpeg で PCM を抽出し Canvas にレンダリング |
| C3 | 音量 / ミュート / ソロ | トラック単位とクリップ単位 |
| C4 | フェードイン / アウト | クリップ両端のフェードハンドル |
| C5 | オーディオ単体インポート | mp3 / wav / m4a |

### D. エフェクト / トランジション
作品の表現力。

| # | 機能 | 概要 |
|---|------|------|
| D1 | クロスフェード / フェード遷移 | クリップ境界にトランジションを配置 |
| D2 | 基本フィルタ | 明るさ / コントラスト / 彩度 / ガンマ |
| D3 | トランスフォーム | 拡大縮小 / 位置 / 回転 (Preview に合成) |
| D4 | クロップ | 画面トリミング |
| D5 | 再生速度変更 | 0.25x〜4x、オーディオピッチ保持オプション |
| D6 | キーフレームアニメーション | 上記パラメータを時間変化で制御 |

### E. テキスト / グラフィックス
タイトル・字幕。

| # | 機能 | 概要 |
|---|------|------|
| E1 | テキストレイヤー | フォント / 色 / サイズ / 位置。タイムライン上でクリップとして配置 |
| E2 | 画像 / ロゴ挿入 | PNG/JPG オーバーレイ |
| E3 | 字幕 SRT インポート / エクスポート | 外部ファイルと連携 |
| E4 | タイトルテンプレート | プリセットの組み込みテキスト |

### F. エクスポート強化
出力の柔軟性。

| # | 機能 | 概要 |
|---|------|------|
| F1 | エクスポートプリセット | YouTube 1080p / Twitter / Instagram Reels 等 |
| F2 | 解像度 / ビットレート / FPS 指定 | ダイアログで選択 |
| F3 | コーデック選択 | H.264 / H.265 / ProRes |
| F4 | 選択範囲のみエクスポート | In/Out 間のみ書き出し |
| F5 | 進捗表示 | ffmpeg の progress をパース |

### G. 運用・品質
長期的な安定性。

| # | 機能 | 概要 |
|---|------|------|
| G1 | プロキシ編集 | 重い元ファイルから低解像度プロキシを生成して編集、書き出しは元ファイル |
| G2 | メディアビン | 取り込み済みファイルのライブラリパネル |
| G3 | 環境設定ダイアログ | プロキシ / 自動保存 / ショートカットカスタマイズ |
| G4 | ログ / 診断情報 | 問題報告用にエクスポート |

## 実装順序 (フェーズ別)

優先度の判断基準:
- **実用性**: 最低限のワークフロー (開く→編集→保存→エクスポート) が成立するか
- **基盤性**: 後続機能の前提になるか (例: オーディオトラック分離はフェードの前提)
- **難易度**: 技術負債を先に返すほど後が楽

### Phase 1 — MVP の完成 (必須)
**目標**: 「開いて編集してもう一度開ける」ワークフローを完成させる。

1. **A1 プロジェクト保存 / 読み込み** — これがないと全てが揮発する
2. **B1 クリップのトリミング UI** — reducer は既存なので UI 追加のみで済む
3. **B5 再生系ショートカット (Space / J/K/L / I/O)** — 編集体験の核
4. **A4 エラートースト UI** — 基盤として早めに入れる

### Phase 2 — 編集の快適さ
**目標**: タイムライン操作のフラストレーションを解消する。

5. **B2 タイムラインズーム**
6. **B3 スナップ**
7. **B7 クリップのコピー / ペースト / 複製**
8. **B4 リップル削除・挿入**
9. **B6 マーカー**

### Phase 3 — オーディオ対応
**目標**: 映像だけでなく音も編集可能にする。C1 が基盤。

10. **C1 オーディオトラック分離** (型定義に `kind` 追加、Track に種別)
11. **C3 音量 / ミュート / ソロ**
12. **C2 波形表示**
13. **C5 オーディオ単体インポート**
14. **C4 フェードイン / アウト**

### Phase 4 — 表現力アップ
**目標**: 「編集できる」から「作品が作れる」へ。

15. **D1 クロスフェード / フェード遷移**
16. **E1 テキストレイヤー**
17. **D2 基本フィルタ** (明るさ / コントラスト / 彩度)
18. **D3 トランスフォーム (拡大縮小 / 位置)**
19. **D5 再生速度変更**
20. **E2 画像 / ロゴ挿入**

### Phase 5 — エクスポート強化
**目標**: 用途ごとに最適な書き出しを可能に。

21. **F5 進捗表示** (まず UX 改善)
22. **F2 解像度 / ビットレート / FPS 指定**
23. **F1 エクスポートプリセット**
24. **F4 選択範囲のみエクスポート**
25. **F3 コーデック選択**

### Phase 6 — 高度な表現
**目標**: プロレベルの表現に近づける。

26. **D6 キーフレームアニメーション** (大規模。D3/D2 を土台に)
27. **D4 クロップ**
28. **E3 字幕 SRT インポート / エクスポート**
29. **E4 タイトルテンプレート**

### Phase 7 — 運用品質
**目標**: 長時間作業・大量素材に耐える。

30. **A3 自動保存 / クラッシュ復旧**
31. **A2 最近使ったファイル**
32. **G2 メディアビン**
33. **G1 プロキシ編集**
34. **G3 環境設定ダイアログ**
35. **G4 ログ / 診断情報**

## 検証方法

各フェーズ完了時に以下を実施:

1. **型チェック / リント**: `npx tsc --noEmit`, `npx oxlint src/`, `npx biome check src/`
2. **手動 E2E**: `npm start` で起動し、以下のゴールデンパスを確認
   - Phase 1: インポート → 分割 → トリム → 保存 → 再起動 → 読込 → 同じ状態
   - Phase 3: 音声付き動画をインポート → 波形が出る → 音量下げ → エクスポートで反映
   - Phase 4: テキストを乗せてエクスポート → 書き出した MP4 にテキストが写る
3. **ffmpeg 出力検証**: `ffprobe` で解像度/コーデック/長さが意図通りか確認
4. **Playwright E2E** (将来): Phase 2 以降で主要フローを自動化

## 注意事項

- 本ロードマップは「全部やる」前提ではなく、**ユーザーが価値を感じるタイミングで区切れる順序** に整えてある。Phase 1 完了時点で「最低限使えるアプリ」になる
- 各フェーズ内の個別機能に着手する際は、改めて個別の詳細プランを作成すること
- Phase 3 の C1 (トラック種別分離) は後続すべての音声機能の前提なので、データモデル変更の影響範囲に注意
